# Angkou-Vps 开发计划

更新时间：2026-06-19

## 一、当前定位

`akvps` 是运行在 VPS 本机里的中文轻量工具。当前目标不是做本地多 VPS 总控，而是让一台 VPS 能快速完成：

- 基础开荒和稳妥调优
- 3x-ui 代理节点部署
- 接入 3x-ui 主控面板
- 自动生成主控入站和客户端
- Komari 探针或主站部署
- 输出可复制的接入卡片和报告

## 二、当前已完成

- 支持 Debian 11/12/13、Ubuntu 22.04/24.04。
- 支持一键导入主控、Cloudflare、节点、Komari 信息。
- 支持 Cloudflare DNS 自动解析，小灰云。
- 支持 SSH 公钥登录、Swap、UFW、系统调优。
- 支持安装或复用 3x-ui。
- 支持主控节点自动添加、更新、验证。
- 支持自动创建远端 VLESS Reality 入站和客户端。
- 支持 HY2 与 HY2 端口跳跃入站结构。
- 支持把主控生成的入站同步到节点本机。
- 支持客户端绑定目标入站，并清理错误绑定。
- 支持验证客户端分享链接和订阅链接。
- 支持 Komari agent 自动发现接入。
- 支持完整清除本机环境。
- 已实测 VLESS Reality 代理可用、流量可回传。

## 三、当前已知问题

- 3x-ui 客户端页会缓存入站下拉列表。后台自动创建入站后，页面可能先显示数字 ID；浏览器强制刷新后恢复正常。
- `README.md` 和测试清单需要跟随最新版本继续整理。
- 当前报告保持轻量，只记录部署结果和必要状态，不扩展成订阅或规则使用手册。
- HY2 / HY2 端口跳跃还未做真实客户端连通测试。

## 四、下一阶段轻量收口

### 1. 3x-ui 主控面板边界

主控面板由昂寇自行登录网站维护，脚本不做复杂面板巡检或自动修复。

- 订阅设置：启用普通 VLESS 订阅，用于 v2rayN / V2Ray 类客户端和昂寇自建 Clash 订阅取节点。
- 订阅设置：启用 Clash/Mihomo 订阅，仅作为不带规则的朋友用订阅或备用订阅。
- 订阅设置：暂不启用 JSON 订阅，除非后续有明确客户端需要。
- 订阅路径：不要使用默认 `/sub/`；普通订阅和 Clash 订阅都应改成随机路径。
- 订阅规则：3x-ui 不写规则；普通订阅和 Clash 订阅都只负责提供节点。
- 出站设置：保持默认直连出站；只在需要封禁、兜底或链式代理时再配置。
- Xray 路由：仅用于服务端兜底，例如禁 BT、拦截内网目标、按入站特殊转发。

### 2. 代理规则与使用策略

规则不进入 `akvps` 主流程：

- 3x-ui 定位为节点、入站、客户端、订阅链接、流量统计和服务端兜底规则。
- Clash/Mihomo 用户在 Clash/Mihomo 配置里维护规则、分组和规则集。
- 昂寇自用沿用现有远端 Mihomo 订阅，并从 3x-ui 普通 VLESS 订阅取节点。
- v2rayN / V2Ray 用户在客户端里选择或维护路由规则。
- 3x-ui Clash 订阅仅作为朋友使用或备用节点来源，不带规则。

### 3. 主控页面缓存提示

- 在报告里明确提示：若客户端关联入站显示数字，按 `Command + Shift + R` 强制刷新。
- 在验证接入完成后也输出同样提示。

### 4. HY2 实测

- 后续需要单独测试 HY2 和 HY2 端口跳跃。
- 测试项包括：客户端能否识别链接、端口是否可达、能否实际访问外网、流量能否回传到 3x-ui。

### 5. 验收报告保持轻量

- 报告只保留部署结果和必要状态，不展开具体订阅使用说明。
- 订阅链接、规则、客户端导入等操作以 3x-ui 主控网站为准，不在脚本报告里做复杂引导。

### 6. 文档更新

- 更新 `README.md` 的菜单和功能描述。
- 更新 `AKVPS-从零实测清单.md` 到当前版本。
- 把常见误判写进排障：客户端关联入站显示数字，是 3x-ui 前端缓存。

### 7. 高级功能只记录，不进入近期主线

- 后续可单独研究 3x-ui / Xray 服务端链式出站自动分流，例如用户连接香港入站，但 YouTube 等指定流量转发到日本上游出站。
- 链式出站自动分流不作为默认方案，需要单独评估延迟、稳定性、上游节点维护、规则影响范围和排障复杂度。
- Token 轮换由昂寇自行处理，脚本不做额外提醒系统。
- 脚本继续保持敏感信息默认遮挡，不在报告里输出完整 Token。

## 五、暂不做

- 不做本地多 VPS 资产总控。
- 不做复杂计费和成本台账。
- 不做 Halo、网站建站、内容系统。
- 不直接替代专业代理客户端的完整规则引擎。
- 不做主控面板设置巡检和自动修复。
